# Active Decisions

- **Architecture**: Adopted a Monorepo structure to simplify version control and deployment pipelines.
- **Backend Hosting**: Selected Railway's $5/mo plan to avoid cold starts and ensure high availability for the Python API.
- **Frontend Hosting**: Vercel selected for the Next.js dashboard due to optimized global edge performance.
- **Database Engine**: Transitioned to PostgreSQL on Railway to guarantee data persistence, replacing ephemeral SQLite.
- **Business Model / Pricing**: Adopted a 3-tier structure (Basic, Scale, Growth). Removed the free tier to monetize immediately.
- **Credential Architecture**: A single user account can now hold multiple distinct Google Ads credentials (Customer IDs), restricted by their subscription tier (1, 3, or Unlimited).
- **Python Constraints**: Pinned backend Python version to 3.11 for Railway deployments to retain the `crypt` module required by `passlib`.
- **Google Ads API Versioning**: `google-ads` library must be kept at v31+ to avoid `501 GRPC` errors from sunset API versions. Env vars for OAuth in `google_ads_service.py` must use `GOOGLE_ADS_CLIENT_ID` / `GOOGLE_ADS_CLIENT_SECRET` (matching `auth_google.py`).
- **Autopilot Architecture (Hybrid Model)**: Adopted a split execution model for the Google Ads orchestrator. Budget protection rules (PAUSE) are executed automatically via a background cron job. Scaling rules (SCALE) generate pending logs that require manual UI approval from the client. Additionally, a "Learning Phase" heuristic (minimum spend > 1.5x Target CPA) protects new campaigns from being shut down prematurely.
- **Domain Override**: Unconditionally force custom domain (googlemaker.thequantpartners.com) in CORS allowed origins to bypass Railway env variable overrides.
- **Agent Workflow**: External AI agents (e.g., Jules) do not push directly to `main`. They always submit their work via Pull Requests for safety and human review. Agents reviewing code must check remote branches.
- **Payment Gateway**: Integrated Lemon Squeezy as Merchant of Record (MoR) for subscription handling, replacing local mock tier assignments.
- **UI Architecture**: Selected Tailwind CSS v3 and Recharts for the primary frontend dashboard aesthetics, prioritizing a high-end dark mode, glassmorphism, and neon gradient design language over standard CSS modules. All new components must be mobile-responsive by default, utilizing sliding sidebars and hamburger menus for navigation.
- **Localization**: Client Dashboard and Landing Page default to English. Landing Page supports EN/ES toggle. SuperAdmin dashboard remains in Spanish. Plan benefits are positioned for Do-It-Yourself (DIY) automation rather than human-supported service.
- **Database Migrations & Production Env**: The local frontend environment (`.env.local`) points to the PRODUCTION API on Railway (`googlemaker-production.up.railway.app`), not `localhost:8000`. Any database schema changes must be accompanied by explicit migration logic (e.g., raw SQL in `lifespan` in `main.py`) and MUST be pushed to `main` so Railway deploys them. Otherwise, testing the local frontend will result in 500 API errors from the un-migrated production database.
- **Pixel / Campaign Mapping**: Instead of attempting to parse complex Account-Level versus Campaign-Level goals in the Google Ads API, the "Manage Pixels" feature determines which campaigns a pixel is anchored to by looking at historical data (`metrics.conversions > 0` over `LAST_30_DAYS` segmented by `conversion_action`). This provides marketers with practical "active" data rather than theoretical configuration.
- **Branding**: Dropped the 'Google' trademark from the product name to prevent legal, OAuth, and payment gateway issues. The brand is officially 'GMaker'.
- **Ideal Customer Profile (ICP)**: Product marketing and default templates are hyper-focused on **US Immigration Lawyers** (targeting the US Latino market).
- **Ad-Spend Based Pricing**: Pricing tiers are gated by managed Google Ads spend volume rather than feature access. All users get the full Autopilot core.
- **Soft Pause Policy**: When a user exceeds their tier's ad-spend limit, GMaker will **NEVER** pause their actual Google Ads campaigns. Instead, GMaker will execute a "Soft Pause", disabling its own Autopilot optimizations and displaying a warning to upgrade, protecting the client's underlying business operations from accidental disruption.
- **Legal & Compliance**: Overhauled all legal clauses (Terms, Privacy, Refunds) specifically tailored to the litigious US Immigration Lawyer market. Added strict liability caps, ad-spend waivers, indemnification, and mandatory arbitration clauses to protect the company.
